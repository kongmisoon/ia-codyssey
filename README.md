## GenAI 기초 1: LLM 기반 업무 자동화 답안 초안

### 2. 시스템 설계 문서

타겟 사용자는 사내 실무자이다. 입력 템플릿은 다음과 같다. [업무 과업] 회의록 요약, [원문] (회의 내용), [원하는 결과] 결정사항/Action Items/리스크/후속 일정, [톤] 간결하고 사내 공유용, [금지] 추측성 표현과 실명 대신 역할명 사용, [확인 질문 규칙] 정보 부족 시 최대 3개까지 질문.

페르소나는 프로젝트 매니저 출신 업무 자동화 코치로 정의하며, 정확성을 우선하고 추측을 금지한다.

시스템 프롬프트 초안: 당신은 프로젝트 매니저 출신 업무 자동화 코치입니다. 회의 내용을 바탕으로 결정사항, Action Items(담당자/기한 포함), 리스크, 후속 일정을 정리하십시오. 정보가 부족하면 최대 3개까지 확인 질문을 먼저 하십시오. 실명 대신 역할명을 사용하고, 확실하지 않은 내용은 확인 필요로 표시하십시오. 최종 답변은 템플릿 형식을 반드시 지키십시오.

Few-shot 예시는 최소 3개를 준비하며, 그중 1개는 모호한 입력 케이스로 구성한다. v1(간단) 프롬프트는 아래 회의 내용을 요약해줘 이고, v2(단계적) 프롬프트는 먼저 목표 출력 항목을 확인하고, 부족하면 질문한 뒤, 템플릿 형식으로 최종 출력하라 이다.

### 3. 실행 로그

실제 대화를 10턴 이상 수행하며, 중간에 톤이나 대상 변경 요청을 1회 포함시킨다. 문제 발생 지점(환각, 문맥 끊김 등)과 수정 결과를 요약해 기록한다. 환각 검증을 위해 사실 질문 5개 이상을 준비하고, 기대 정답과 근거, Pass 또는 Fail 판정을 함께 기록한다.

위 항목은 초안 템플릿이며, 실제 모델 테스트 결과와 대화 로그로 괄호 부분을 채워 완성해야 한다.
